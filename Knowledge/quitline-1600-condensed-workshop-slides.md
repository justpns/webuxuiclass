# Quitline 1600 Condensed Workshop Slides

## Slide 1 — Assignment Workshop
**Quitline 1600: Mobile-First Website Redesign**

Today’s task:

- simplify a complex brief
- separate UX and UI work
- design a clearer service experience

Main goal:
Help users understand the service and choose the right action quickly.

---

## Slide 2 — Project Framing
This is a **public service website case study**.

We are not building:

- full backend systems
- real dashboards
- full CMS logic
- real PDPA implementation

We are designing:

- the user-facing website experience
- mobile-first service access
- clearer information flow

---

## Slide 3 — Core User Actions
The homepage should make these actions obvious:

- Call 1600
- Add LINE OA
- Talk to QuitBot
- Start an assessment

The key UX question:

**How do we get users to help in 1-2 steps?**

---

## Slide 4 — UX Side Workshop
**UX team focus**

Solve:

- what users need first
- what content belongs on the homepage
- what should move to secondary pages
- how the path to help becomes shorter

UX outputs:

- problem summary
- target users
- sitemap
- user journey
- wireframes

---

## Slide 5 — UX Success Check
Your UX should make the experience feel:

- understandable
- simple
- guided
- low-friction

Ask:

- Is the homepage too crowded?
- Is the primary CTA clear?
- Can new users understand the next step immediately?

---

## Slide 6 — UI Side Workshop
**UI team focus**

Solve:

- what users notice first
- how CTA hierarchy is shown visually
- how the page builds trust
- how mobile layout improves readability

UI outputs:

- visual direction
- homepage UI
- service options UI
- one content page UI

---

## Slide 7 — UI Success Check
Your UI should feel:

- modern
- trustworthy
- calm
- easy to scan

Ask:

- Is the primary action visually strongest?
- Does the design reduce clutter?
- Is the screen readable on mobile?

---

## Slide 8 — Suggested Homepage Structure
Recommended blocks:

1. Hero section
2. Main CTA group
3. Support options overview
4. Quick assessment entry
5. Knowledge Hub preview
6. Reassurance or success story

Rule:

**Do not put everything on the homepage.**

---

## Slide 9 — Deliverables
Each team submits:

- UX summary
- simplified sitemap
- user journey
- wireframes
- final UI screens
- short design rationale

Optional:

- clickable prototype

---

## Slide 10 — Final Brief
Use this statement:

> Redesign the Quitline 1600 website as a mobile-first UX/UI concept that helps users quickly understand the service and choose the right support pathway with minimal confusion.

Final reminder:

**Do not design everything. Design what matters most.**
