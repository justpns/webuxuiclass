# Student Assignment Sheet

## Assignment Title
**Quitline 1600: Mobile-First Website Redesign Concept**

## Assignment Type
UX/UI case study project

## Background
Quitline 1600 is a support service for people who want to quit smoking. The current brief describes a large digital platform with many features, services, and system requirements.

For this class, we will not build the full real-world platform. Instead, you will use the brief as a **case study** and redesign the **user-facing website experience** so it becomes clearer, easier to use, and more effective on mobile devices.

This assignment is about simplifying complexity.

---

## Assignment Goal
Design a **mobile-first website concept** that helps users quickly understand the service and choose the right next step.

Your design should help users do one or more of these actions easily:

- call 1600
- add LINE OA
- talk to QuitBot
- start a basic assessment
- explore useful quitting support content

---

## Learning Objectives
By completing this assignment, you should be able to:

- analyze a complex design brief
- identify key users and their needs
- simplify information into a usable structure
- design clear CTA hierarchy
- create a mobile-first page flow
- develop wireframes or UI screens based on user goals
- explain your design decisions clearly

---

## Your Task
Create a redesign concept for the Quitline 1600 website as a **frontend UX/UI project**.

You should focus on:

- homepage clarity
- service access
- information architecture
- mobile-first usability
- visual hierarchy
- trust and ease of use

You do **not** need to build:

- a real CMS
- a real dashboard
- a working chatbot system
- a full backend
- a real PDPA management system

You only need to design the **user-facing website experience**.

---

## Suggested Content Structure

### Homepage should clearly show:

- what Quitline 1600 is
- who the service is for
- what users should do next
- the main support options

### Recommended homepage blocks:

1. Hero section
2. Main CTA buttons
3. Short explanation of support options
4. Quick assessment or “start here” section
5. Knowledge Hub preview
6. Trust/reassurance section
7. Footer or contact area

---

## Design Requirements

Your work should demonstrate:

- mobile-first thinking
- clear hierarchy
- readable typography
- simple navigation
- visible CTA actions
- supportive, trustworthy tone
- reduced cognitive overload

Your design should help users reach help in **1-2 steps** whenever possible.

---

## Process Requirements
You must submit evidence of process, not only final screens.

Include:

1. Brief summary
2. Target users
3. Key problems in the current experience
4. Simplified sitemap
5. Key user journey
6. Wireframes
7. Final UI screens or prototype
8. Short rationale

---

## Deliverables
Submit the following:

1. **Problem Summary**
   Explain the main issues in the current brief in simple terms.

2. **Target User Summary**
   Identify the main user groups and what they need most.

3. **Simplified Sitemap**
   Show the core page structure for your redesigned website.

4. **User Journey**
   Show how one main user moves from landing on the site to getting help.

5. **Wireframes**
   Create low-fidelity wireframes for your main pages.

6. **Final UI Design**
   Create polished screen designs for the required pages or sections.

7. **Design Rationale**
   Write a short explanation of why you made your major design decisions.

8. **Optional Clickable Prototype**
   If possible, connect the screens into a simple prototype.

---

## Suggested Workflow

1. Read the brief carefully
2. Identify the most important user goals
3. Reduce the brief into essential features
4. Build a simple sitemap
5. Sketch mobile-first wireframes
6. Design final UI screens
7. Review whether users can find help quickly

---

## Constraints

- Focus on the frontend experience only
- Do not over-design unnecessary pages
- Do not include advanced system diagrams unless requested
- Keep content clear and concise
- Avoid overcrowded homepage layouts
- Prioritize usability over decoration

---

## Evaluation Criteria

Your work will be evaluated based on:

1. **Clarity of the core message**
   Can users quickly understand what the service is?

2. **CTA hierarchy**
   Are the main actions obvious and easy to choose?

3. **Mobile-first usability**
   Does the layout work well for mobile users?

4. **Information architecture**
   Is the content structure simple and logical?

5. **Wireframe quality**
   Do the layouts solve the user problem clearly?

6. **UI execution**
   Does the visual design support trust, clarity, and ease of use?

7. **Design reasoning**
   Can you justify your choices based on user needs?
