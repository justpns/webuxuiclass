# Website UX/UI Mini Case Study Block

## Slide 1 — Case Study Setup
**Real websites: UX is not the same as UI**

We will compare 4 live websites:

- Apple
- GOV.UK
- Craigslist
- Berkshire Hathaway

Goal:

- identify good UX
- identify good UI
- separate usefulness from visual style

Main lesson:

**Beautiful does not always mean usable. Plain does not always mean bad.**

---

## Slide 2 — Good UX + Good UI
**Examples: Apple and GOV.UK**

### Apple
- strong visual hierarchy
- clear CTA flow
- clean scanning path
- premium UI that supports the product story

### GOV.UK
- plain but highly usable
- strong search and task structure
- direct language
- service-first navigation

Teaching point:

**Good UI can support UX, but strong UX does not require flashy design.**

---

## Slide 3 — Bad UI but Good UX
**Example: Craigslist**

- visually outdated
- weak aesthetics
- little visual polish
- but still fast and task-oriented
- strong category structure
- familiar repeat-user flow
- low friction for posting and browsing

Teaching point:

**Users can tolerate ugly interfaces when the system is easy to use and helps them finish the task quickly.**

---

## Slide 4 — Bad UX + Bad UI
**Example: Berkshire Hathaway**

- weak visual hierarchy
- outdated layout
- dense wall of links
- poor first-time guidance
- little support for scanning or prioritization

Teaching point:

**If users must work too hard to understand the page, both UX and UI are failing together.**

Final question for students:

> Which matters more first: visual beauty, or clarity of action?
