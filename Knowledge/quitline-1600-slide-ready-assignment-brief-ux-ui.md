# Quitline 1600 Slide-Ready Assignment Brief

## Slide 1 — Workshop Assignment
**Quitline 1600: Mobile-First Website Redesign Concept**

Class workshop:

- UX side
- UI side

Goal:
Turn a complex real-world brief into a clear, usable, mobile-first website concept.

---

## Slide 2 — Why This Project
This is a **public service website**, not just a marketing page.

Users may arrive with:

- uncertainty
- stress
- low motivation
- limited time
- mobile-only access

So the website must feel:

- clear
- supportive
- immediate
- easy to act on

---

## Slide 3 — The Real Design Problem
The original brief is too large for one class project.

It includes:

- service access
- content organization
- chatbot entry
- assessments
- dashboards
- CMS
- SEO
- PDPA

In class, we will **reduce the scope** and focus on the **user-facing website experience**.

---

## Slide 4 — Classroom Scope
For this workshop, you are **not** building the full platform.

You are designing:

- a better homepage
- clearer service access
- simpler information structure
- a more usable mobile experience

Think:

**prototype concept, not full production system**

---

## Slide 5 — Main User Goal
Users should quickly understand:

- What is Quitline 1600?
- Who is it for?
- What should I do next?

The main actions should be obvious:

- Call 1600
- Add LINE OA
- Talk to QuitBot
- Start an assessment

---

## Slide 6 — Required Output
Each team or student will create a **mobile-first redesign concept**.

Minimum required pages or sections:

1. Homepage
2. Service options / support entry
3. One knowledge or content page

Optional extra:

- About page
- FAQ
- Assessment entry
- Success story page

---

## Slide 7 — Workshop Split
This assignment has **2 sides**:

### UX Side
Focus on:

- user goals
- information architecture
- user flow
- homepage logic
- wireframes

### UI Side
Focus on:

- visual hierarchy
- layout
- typography
- CTA emphasis
- trust and tone

---

## Slide 8 — UX Workshop Brief
**UX Side Question:**

How do we simplify this service so users can get help with as little thinking as possible?

Your UX job is to define:

- what users need first
- what content matters most
- what should be removed or moved
- how users move from landing to action

---

## Slide 9 — UX Tasks
Create these UX outputs:

1. Problem summary
2. Target user summary
3. Simplified sitemap
4. One key user journey
5. Mobile-first wireframes

Focus on:

- reducing cognitive overload
- shortening the path to help
- making choices easier

---

## Slide 10 — UX Questions to Answer
Your UX solution should answer:

- What must users see first?
- Which CTA is primary?
- How many choices should appear on the homepage?
- What content belongs on the homepage?
- What should move to secondary pages?
- Can users reach help in 1-2 steps?

---

## Slide 11 — UX Success Criteria
Strong UX work should show:

- clear homepage structure
- simple navigation
- logical grouping of content
- obvious next actions
- user-friendly mobile flow

If the user lands here for the first time, the experience should feel:

- understandable
- manageable
- action-oriented

---

## Slide 12 — UI Workshop Brief
**UI Side Question:**

How do we visually guide attention so users notice the right action immediately?

Your UI job is to translate the UX structure into a visual design that feels:

- modern
- trustworthy
- calm
- supportive
- easy to scan

---

## Slide 13 — UI Tasks
Create these UI outputs:

1. Visual direction or mood
2. Homepage UI design
3. Service options page UI
4. One content page UI
5. Mobile-first responsive layout thinking

Focus on:

- hierarchy
- readability
- button prominence
- spacing
- visual clarity

---

## Slide 14 — UI Questions to Answer
Your UI solution should answer:

- What do users notice first?
- Are primary CTAs visually stronger than secondary actions?
- Does the page feel friendly without feeling childish?
- Does the design build trust?
- Is the page easy to scan on mobile?
- Is the interface cleaner than the current concept?

---

## Slide 15 — UI Success Criteria
Strong UI work should show:

- strong visual hierarchy
- readable typography
- good spacing
- clear CTA styling
- supportive color use
- modern but accessible layout

The interface should feel:

- simple
- reliable
- human
- focused

---

## Slide 16 — Suggested Homepage Structure
Recommended homepage blocks:

1. Hero section
2. Main CTA group
3. Support options overview
4. Quick assessment entry
5. Knowledge Hub preview
6. Reassurance or success story
7. Footer / contact

---

## Slide 17 — What Not to Overbuild
Do not spend workshop time on:

- full CMS logic
- backend systems
- dashboard implementation
- real chatbot flow logic
- detailed PDPA architecture
- SEO technical setup

Acknowledge them if needed, but do not let them distract from the design task.

---

## Slide 18 — Deliverables
Submit:

1. UX summary
2. Sitemap
3. User journey
4. Wireframes
5. Final UI screens
6. Short design rationale

Optional:

- clickable prototype

---

## Slide 19 — Evaluation
You will be evaluated on:

### UX
- clarity of structure
- logic of flow
- reduction of complexity
- quality of wireframes

### UI
- strength of hierarchy
- CTA visibility
- readability
- mobile-first design quality

### Overall
- how well the design helps users get support quickly

---

## Slide 20 — Final Assignment Statement
Use this as your team brief:

> Redesign the Quitline 1600 website as a mobile-first UX/UI concept that helps users quickly understand the service and choose the right support pathway with minimal confusion.

Final reminder:

**Do not design everything. Design what matters most.**
