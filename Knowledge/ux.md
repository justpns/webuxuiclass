UX PRINCIPLES (Experience, Flow, Structure, Behavior)
1. Don’t Make Users Think
Core Idea

Interfaces should feel obvious and self-explanatory.

Users should instantly understand:

Where they are
What they can do
What is clickable
What happens next
Bad UX
Confusing labels
Hidden actions
Complex flows
Unclear navigation
Good UX
Predictable patterns
Clear hierarchy
Familiar layouts
Obvious actions
Modern AI Era Interpretation

AI should reduce friction, not add mystery.
Users should still understand:

What AI is doing
Why results appear
What action to take next
2. Users Scan, Not Read
Core Idea

People rarely read websites word-by-word.

They:

Scan headings
Look for keywords
Jump between sections
Ignore large text blocks
UX Implications

Design content for scanning:

Clear headings
Short paragraphs
Bullet points
Visual grouping
Highlighted keywords
Mobile-First Note

Scanning behavior is stronger on mobile.

3. Users “Satisfice”
Meaning

Users choose the first option that seems “good enough.”

They do not always search for the best solution.

UX Implication

Make the best path:

Most visible
Simplest
Fastest
Example

Primary CTA should dominate visually.

Bad:

5 equal buttons

Good:

1 clear primary action
4. Users Don’t Figure Things Out
Core Idea

Most users do not stop to learn systems.

If confused:

They guess
Skip
Leave
UX Principle

Remove unnecessary learning.

Design Goal

Recognition > Recall

Users should recognize actions instantly.

5. Create Clear Visual Hierarchy
Purpose

Visual hierarchy guides attention.

Strong Hierarchy Includes
Size contrast
Spacing
Typography weight
Color emphasis
Positioning
Users Should Instantly See
What matters most
What action to take
Content relationships
6. Navigation Must Be Obvious
Navigation Should Answer
Where am I?
Where can I go?
Where have I been?
UX Rules
Keep nav consistent
Use familiar naming
Avoid creative/confusing labels
Show active states
Include breadcrumbs if needed
Modern UX

Sticky navigation improves usability on long pages.

7. Minimize Cognitive Load
Cognitive Load = Mental Effort

Users already spend energy:

Comparing
Deciding
Reading
Trusting

Design should reduce extra effort.

Reduce By
Simplifying choices
Reducing steps
Auto-filling data
Using defaults
Chunking information
8. Every Click Must Feel Worthwhile
Core Idea

Each interaction should feel:

Predictable
Useful
Fast
Bad Experience

User clicks and gets:

Confusion
Delay
Unexpected result
Good Experience

User gains:

Progress
Confidence
Clear next step
9. Eliminate Unnecessary Choices
Based on Decision Fatigue

Too many choices reduce action.

UX Strategy

Curate options.
Guide users.

Example

Instead of:

20 pricing options

Use:

Recommended plan
3 simple tiers
10. Make Important Things Visually Prominent
Users Notice:
Large objects
Contrast
Motion
Isolation
Color differences
Important Actions Should Stand Out

Examples:

Buy Now
Book Now
Contact
Start Free Trial
11. Use Conventions
Conventions Reduce Learning

Users expect:

Logo top-left
Cart top-right
Underlined links
Hamburger menu on mobile
Principle

Do not reinvent familiar patterns unless clearly better.

12. Home Page Purpose
Homepage Must Answer Quickly
What is this?
Why should I care?
What can I do here?
Homepage Should
Build trust
Show value proposition
Guide next actions
Reduce uncertainty
13. Reduce Noise
Every Element Competes for Attention

Too much:

Text
Motion
Colors
Popups
Buttons

Creates cognitive overload.

Good UX Removes:
Redundant content
Decorative clutter
Unnecessary interactions
14. Content is Part of UX
Good Writing = Better UX

Use:

Plain language
Short sentences
Clear CTAs
Human wording
Avoid
Corporate jargon
Long introductions
Complex terminology
15. Mobile UX Requires Prioritization
Mobile Forces Simplicity

Important content must appear first.

Mobile UX Rules
Large tap targets
Thumb-friendly layouts
Reduced scrolling friction
Clear CTA visibility
Fast loading
16. Usability Testing Is Essential
Steve Krug’s Key Principle

Testing with even a few users reveals major problems.

Observe:
Confusion points
Navigation failures
Hesitation
Misclicks
Important Insight

You are NOT the user.
