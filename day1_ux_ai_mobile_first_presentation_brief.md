# Codex Brief: Mobile-First HTML Presentation

## Project Title
**Day 1: Website Design Fundamentals — UX**  
**Theme:** Don’t Make Users Think in the AI Era

## Goal
Create a **mobile-first HTML-based presentation** for a full-day master’s level teaching session from **09:00–17:00**.

The presentation should help students learn UX fundamentals using the philosophy of **“Don’t Make Me Think”** combined with modern **AI-assisted UX workflow**.

The final output should be a polished, responsive, browser-based slide deck that works beautifully on mobile, tablet, and desktop.

---

# Core Teaching Philosophy

The whole presentation must reinforce:

- Users scan, they do not read everything.
- Clear design beats clever design.
- Navigation should feel obvious.
- Every page should answer: “What is this?” “Who is it for?” “What should I do next?”
- AI can generate ideas, but humans must simplify, judge, and decide.
- Good UX reduces cognitive load.

Main message:

> AI can generate. Designers must decide.

---

# Technical Requirements

## Output
Create a complete static HTML presentation with:

- `index.html`
- `style.css`
- `script.js`
- Optional `/assets` folder if needed

## Design Direction
Use a premium modern education/workshop style:

- Mobile-first layout
- Clean editorial typography
- Strong visual hierarchy

- Card-based teaching blocks
- Smooth transitions
- Dark/navy + white + electric blue accent style
- Modern AI/workshop feeling
- Large readable text for classroom projector
- Comfortable mobile reading

## Responsive Behavior
The presentation must work as:

- Mobile: vertical swipe/scroll-friendly cards
- Tablet: centered slide panels

Use CSS media queries.

## Navigation
Include:

- Previous / Next buttons
- Keyboard navigation using arrow keys
- Mobile swipe support if possible
- Slide progress indicator
- Current section indicator
- Optional slide number

## Interaction Style
Keep interactions simple:

- Click/tap to continue
- Smooth slide transition
- Optional activity timer buttons
- Optional “show prompt” collapsible cards

Do not overbuild. Prioritize clarity and usability.

## Section 3: Information Architecture
Time: 10:45–12:00  
Slides: 25–41

### Slide 25 — Section Divider
**Information Architecture**  
Make navigation obvious.

### Slide 26 — What Is Information Architecture?
Definition:

Information architecture is how content is organized so users can find what they need.

### Slide 27 — Why Navigation Matters
Navigation answers:

- Where can I go?
- Where am I now?
- What is available here?

### Slide 28 — Bad Navigation Patterns
Examples:

- too many menu items
- vague labels
- hidden important pages
- duplicated categories
- clever names that users do not understand

### Slide 29 — Good Navigation Principles
Navigation should be:

- obvious
- predictable
- short
- grouped logically
- written in plain language

### Slide 30 — Decision Fatigue
More choices often create slower decisions.

Use simple visual: 3 choices vs 15 choices.

### Slide 31 — Hick’s Law
Simple explanation:

The more choices users have, the longer they take to decide.

Formula display optional:

```text
Decision Time = increases as number of choices increases
```

Do not overexplain math.

### Slide 32 — CTA Hierarchy
Explain:

Not every button has the same importance.

Levels:

- Primary CTA
- Secondary CTA
- Supporting link

### Slide 33 — Example: Bad CTA Hierarchy
Show a mock hero with too many equal buttons.

### Slide 34 — Example: Better CTA Hierarchy
Show one strong primary CTA and one secondary link.

### Slide 35 — Mobile Navigation
Mobile navigation must be even simpler because:

- screen is small
- attention is lower
- thumb movement matters
- users leave faster

### Slide 36 — AI Prompt: Sitemap
Prompt card:

### Slide 37 — AI Prompt: Simplify Navigation

### Slide 38 — Human vs AI Sitemap

### Slide 39 — Workshop: Create Sitemap

### Slide 41 — Section Key Takeaway
